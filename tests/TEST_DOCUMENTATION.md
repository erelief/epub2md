# EPUB到Markdown转换器 - 集成测试文档

## 测试概述

本文档描述了为EPUB到Markdown转换器创建的完整集成测试套件。测试套件验证了系统的端到端功能，确保所有组件协同工作，并测试了各种EPUB格式和边缘情况。

## 测试文件结构

```
├── test_integration.py      # 主要集成测试
├── test_edge_cases.py       # 边缘情况和错误处理测试
├── run_all_tests.py         # 完整测试套件运行器
└── TEST_DOCUMENTATION.md    # 本文档
```

## 集成测试 (test_integration.py)

### 测试覆盖范围

#### 1. 完整转换流程测试
- **test_complete_conversion_chinese**: 测试中文EPUB的完整转换流程
- **test_complete_conversion_english**: 测试英文EPUB的完整转换流程
- 验证语言检测、文件生成、内容转换的完整性

#### 2. 图片处理测试
- **test_image_extraction_and_linking**: 测试图片提取和链接更新
- 验证图片文件被正确提取到输出目录
- 验证Markdown中的图片链接被正确更新为相对路径

#### 3. 注释处理测试
- **test_annotation_processing**: 测试注释内联化功能
- 验证跨文件注释链接被转换为内联注释
- 验证中文注释使用"（注：...）"格式，英文使用"(Note: ...)"格式

#### 4. 封面处理测试
- **test_cover_handling**: 测试封面图片处理
- 验证封面图片被保存为"00_书名_cover.扩展名"格式
- 验证封面页不生成对应的Markdown文件

#### 5. 输出结构测试
- **test_output_directory_structure**: 测试输出目录结构
- 验证输出目录在EPUB文件同目录下创建
- 验证目录名与EPUB文件名一致

#### 6. 文件验证测试
- **test_file_validation**: 测试EPUB文件验证功能
- 验证有效EPUB文件被正确识别
- 验证无效文件被拒绝

#### 7. Unicode处理测试
- **test_unicode_handling**: 测试Unicode字符处理
- 验证中文字符被正确处理和保存
- 验证Unicode目录名被正确处理

#### 8. 错误处理测试
- **test_error_handling**: 测试基本错误处理
- 验证损坏的EPUB文件被正确拒绝
- 验证不存在的文件产生适当的错误

#### 9. 组件集成测试
- **test_component_integration**: 测试所有组件协同工作
- 验证EPUB解析器、语言检测器、封面处理器、内容处理器、图片处理器等所有组件正常协作

#### 10. 性能测试
- **test_performance_and_timing**: 测试性能和计时
- 验证处理时间被正确记录
- 验证转换在合理时间内完成

### 测试数据生成

测试套件动态生成各种类型的EPUB文件：

1. **中文EPUB** (`chinese_simple`): 包含中文内容的基本EPUB
2. **英文EPUB** (`english_simple`): 包含英文内容的基本EPUB
3. **包含图片的EPUB** (`with_images`): 测试图片提取和链接处理
4. **包含注释的EPUB** (`with_annotations`): 测试注释内联化功能
5. **包含封面的EPUB** (`with_cover`): 测试封面处理功能

## 边缘情况测试 (test_edge_cases.py)

### 测试覆盖范围

#### 1. 空EPUB测试
- **test_empty_epub**: 测试没有内容文件的EPUB
- 验证系统正确处理并报告错误

#### 2. 格式错误的HTML测试
- **test_malformed_html_content**: 测试包含格式错误HTML的EPUB
- 验证系统能够处理未闭合标签、嵌套错误等问题
- 验证系统不会因HTML错误而崩溃

#### 3. 特殊字符测试
- **test_special_characters_in_filenames**: 测试文件名中的特殊字符
- 验证Windows非法字符被正确替换
- 验证生成的文件名符合文件系统规范

#### 4. 长内容测试
- **test_very_long_content**: 测试非常长的内容处理
- 验证系统能够处理大量文本内容
- 验证内存使用和性能表现

#### 5. 缺失元数据测试
- **test_missing_metadata**: 测试缺少标题、作者等元数据的EPUB
- 验证系统使用合理的默认值
- 验证语言检测回退机制

#### 6. 文件名清理测试
- **test_filename_sanitization**: 测试文件名清理功能
- 验证各种特殊字符、控制字符、保留名称的处理
- 验证Unicode字符的正确处理

#### 7. 无效EPUB结构测试
- **test_invalid_epub_structures**: 测试各种无效的EPUB结构
- 验证缺少必需文件的EPUB被正确拒绝
- 验证损坏的ZIP文件被正确识别

#### 8. 编码问题测试
- **test_encoding_issues**: 测试各种编码和Unicode字符
- 验证中文、日文、韩文、俄文、阿拉伯文等多语言支持
- 验证表情符号和特殊符号的处理

## 测试运行器 (run_all_tests.py)

### 功能特性

1. **统一测试入口**: 运行所有测试套件的单一入口点
2. **详细结果报告**: 提供测试通过率、失败详情等统计信息
3. **时间统计**: 记录测试执行时间
4. **多语言支持**: 中英文双语测试报告
5. **退出代码**: 根据测试结果返回适当的退出代码

### 使用方法

```bash
# 运行完整测试套件
python run_all_tests.py

# 运行单独的测试文件
python test_integration.py
python test_edge_cases.py
```
